# Demo Script

## 2-minute judge demo

1. Open Mono OS Lite.
2. Press `Build a product roadmap for my startup idea`.
3. Show the chat-first UI: project chips, user bubble, Mono response, and text input.
4. Show that the intent is `Product strategy planning`.
5. Point to `Thinking Notes`: observe, classify, compress, index, route, and escalate.
6. Show semantic compression, memory record id, graph rows, visual tags, and multi-agent routing.
7. Press `Move $500 to my savings account`.
8. Show `L3` risk, authentication required, and the approval gate.
9. Press `Authenticate demo gate`.
10. Explain that only a mocked transfer shell executes and the audit log records the full trace.
11. Show that the financial payload is not sent to ChatGPT.

## 5-minute extended demo

1. Run the calendar preset and show low-risk personal admin flow.
2. Run the client email preset and show L2 approval required.
3. Approve the draft and show workflow status changes.
4. Run market research and show simulated ChatGPT escalation after local Gemma parsing.
5. Run startup roadmap and show multi-agent routing across Strategy, Research, Product, and Builder agents.
6. Run the financial transfer and show L3 authentication.
7. Enter a custom command and show deterministic classification still works.

## Exact commands

- `Summarise my calendar tomorrow and prepare my top 3 priorities`
- `Draft a reply to the latest client email and wait for approval`
- `Research this market and create a short action plan`
- `Move $500 to my savings account`
- `Build a product roadmap for my startup idea`
- Custom: `Plan my launch week and prepare a founder checklist`

## What each screen proves

- Projects: conversation/project sidebar for fast judge presets.
- Chat: natural-language input and simulated local Gemma/Ollama response.
- Thinking Notes: visible intent interpretation before compression and indexing.
- Semantic Memory: compressed task packet, memory id, graph-relational rows, and visual context tags.
- Execution: local result, simulated ChatGPT escalation, and multi-agent routing.
- Approval Gate: L2/L3 gates before sensitive execution.
- Audit panel: local traceability for every module.
- Animated demo: `demo/mono-os-lite-demo.svg` gives a browser-playable storyboard when emulator capture is unavailable.
