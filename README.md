# Mono OS Lite

Mono OS Lite is a privacy-first AI operating layer demo for Android. It presents a simple chatbot UI while simulating the deeper operating layer behind the scenes: intent classification, thinking notes, semantic compression, local memory indexing, graph-relational recall, visual context tags, risk gates, approval handling, mock orchestration, local LLM response simulation, ChatGPT escalation simulation, and audit logging.

## Grand Prize positioning

The demo is designed for the Grand Prize track: it looks as approachable as a chatbot, but proves an end-to-end AI OS control layer underneath. Judges can see how a raw user intent becomes structured, risk-aware, multi-agent execution without using real sensitive data, Firebase, or a backend.

## Setup

Prerequisites:

- Android Studio
- JDK 17 or the Android Studio bundled JBR
- Android SDK 36
- Android emulator or connected Android device, API 24+

This Windows shell currently needs `JAVA_HOME` set to Android Studio's JBR:

```powershell
$env:JAVA_HOME='C:\Program Files\Android\Android Studio\jbr'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
```

## Run instructions

Open the repo in Android Studio and run the `app` configuration, or use:

```powershell
.\gradlew.bat installDebug
& "$env:LOCALAPPDATA\Android\Sdk\platform-tools\adb.exe" shell am start -n com.monoos.lite/.MainActivity
```

## Build APK

```powershell
.\gradlew.bat testDebugUnitTest assembleDebug
```

APK output:

```text
app/build/outputs/apk/debug/app-debug.apk
```

## Demo flow

1. Enter a command in the chat box or select a project preset.
2. Read the Mono response generated by the simulated local Gemma/Ollama layer.
3. Review `Thinking Notes` to see observe, classify, compress, index, route, and escalation decisions.
4. Inspect `Semantic Memory` for the compressed intent, local memory record id, graph rows, and visual tags.
5. Inspect `Execution` for agent routing and simulated ChatGPT escalation on complex tasks.
6. For client email and payment flows, use the approval/authentication gate.
7. For `Move $500 to my savings account`, confirm that the initial state is `L3` and the cloud payload is not sent.

## Known limitations

- All integrations are mock-only by design.
- Local Gemma/Ollama behavior and ChatGPT escalation are simulated in-app; no backend or real API key is used.
- Real Ollama integration can be added later through `10.0.2.2:11434` for an emulator or a LAN host for a physical device.
- No real calendar, email, banking, screen, microphone, or speech-to-text data is accessed.
- Local launch was verified on an Android 35 AOSP ATD emulator named `MonoOSLite_API35`.
- The ATD image returned black framebuffer screenshots in this environment even while the app was resumed and interactive. Use a physical Android device or a full Google APIs emulator image for judge-facing visuals.
